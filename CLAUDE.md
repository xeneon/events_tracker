# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Events Tracker ingests events from multiple sources into PostgreSQL and exports them to Google Sheets. It has two interfaces:
- **Web UI** (`app/`) ‚Äî FastAPI + Jinja2, runs via Docker, provides a browser-based control panel
- **CLI** (`ingest/`) ‚Äî standalone, runs directly with `python -m ingest`

## Running with Docker (primary deployment)

```bash
docker compose build
docker compose up -d

# On remote Docker context
docker --context remote-server compose build web
docker --context remote-server compose up -d web
```

Web UI is at `http://localhost:8080`. Postgres runs as a container on a named volume (`pg_data`). Config (API keys) persists on a named volume (`config_data` ‚Üí `/config`). Schema and seed data are created automatically on first startup.

**Do not run `docker compose down -v`** ‚Äî this deletes both volumes (DB and config).

## CLI Usage (standalone, outside Docker)

```bash
python -m ingest --list
python -m ingest calendarific
python -m ingest trakt --dry-run
python -m ingest --all                   # all ingesters + export
python -m ingest.export_sheets           # export only
python -m ingest.export_sheets --dry-run
```

Requires `.env` at project root (or `CONFIG_DIR/.env` in Docker). `DATABASE_URL` must be set; all API keys are optional per-ingester.

## Architecture

### `ingest/` package

Self-contained ingestion library used by both the CLI and the web UI.

- `config.py` ‚Äî pydantic-settings `Settings` singleton. Searches for `.env` starting at `CONFIG_DIR/.env`, then CWD, then walking up. **All submodules do `from .config import settings`** ‚Äî they hold a reference to the same object.
- `db.py` ‚Äî async SQLAlchemy engine and `async_session_maker`, created at import time from `DATABASE_URL`.
- `models.py` ‚Äî `Category`, `DataSource`, `Event` models. Upsert key: constraint `uq_events_source_external` on `(data_source_id, external_id)`.
- `base.py` ‚Äî `BaseIngester` ABC. `run()` calls `fetch_events()` ‚Üí `normalize()` ‚Üí `_apply_log_scale()` ‚Üí `upsert_events()`. All ingesters take `(session, source)` as constructor args.
- `export_sheets.py` ‚Äî `fetch_rows()` returns `(headers, rows)` (column names come from query result, not hardcoded). `write_to_sheet(rows, headers)` uses those headers for the sheet header row and column count.

### `app/` package

FastAPI web UI layered on top of `ingest/`.

- `main.py` ‚Äî FastAPI app with lifespan that calls `seed.run_seed()` (idempotent `create_all` + seed categories/data_sources).
- `seed.py` ‚Äî Seeds 9 categories and 4 data sources with `ON CONFLICT DO NOTHING`.
- `config_store.py` ‚Äî Reads/writes `/config/config.json` and regenerates `/config/.env`. `EXPORT_QUERY` is stored in `config.json` only (not `.env`, because multiline SQL breaks `.env` format). Exposes `_DEFAULT_EXPORT_QUERY` (from `ingest.export_sheets.QUERY.text`).
- `runner.py` ‚Äî `start_run(source_alias)` creates an `asyncio.Queue`, launches a background task, returns a `run_id`. `_QueueHandler` attaches to root logger before each run and removes in `finally`. Runs are serialized via `asyncio.Semaphore(1)`. `stream_run(run_id)` is an async generator yielding raw SSE strings. **Settings reload**: `_reload_settings()` mutates `config_mod.settings` attributes in-place so all submodules see updated values; `DATABASE_URL` comes from the environment and is unaffected.
- `routes/home.py` ‚Äî `GET /`, `POST /run/{source}`, `GET /stream/{run_id}`.
- `routes/config_routes.py` ‚Äî `GET /config`, `POST /config`. Passes `default_query` and `is_custom_query` to template for the reset-to-default UI.
- `templates/` ‚Äî PicoCSS + vanilla JS. SSE log streaming uses `EventSource` on the client, `[DONE]` sentinel closes the stream.

### Settings priority

`DATABASE_URL` is set in `docker-compose.yml` environment (highest priority ‚Äî never overwritten by config reload). All other keys come from `CONFIG_DIR/.env` (generated by `config_store.save_config()`).

## Popularity Scoring

Two fields:
- **`popularity_score`** (Integer) ‚Äî raw metric as-is: Last.fm listeners, IGDB Want-to-Play √ó 1M, Trakt list_count/votes.
- **`impact_level`** (SmallInteger, 0-100) ‚Äî sqrt-scaled relative to category max. Formula: `sqrt(value) / sqrt(max_value) * 100`. Computed by `BaseIngester._apply_log_scale()` after normalization.

## Hardcoded Ingester Parameters

- **Calendarific:** country=`US`, years = current + next
- **Trakt:** `limit=100` for anticipated movies/shows; premiere search window = 180 days; only `returning series` with `episode_number==1 & season>1` (no series premieres)
- **IGDB:** candidates must have `hypes > 0`; `limit=100`; PopScore type 2 (Want to Play) only
- **Wikipedia Albums:** current year only (no next-year support); skips tables/entries with "TBA"

## Key Conventions

- **Calendarific:** Deduplicates ~633 raw entries per year to ~250 using `urlid + date` key, keeping highest `DEDUP_PRIORITY` variant. Uses `primary_type` (not `type` array) for category mapping.
- **IGDB:** Two-step fetch ‚Äî first get games with `hypes > 0`, then batch-fetch Want-to-Play scores, re-sort, and attach scores. Fresh Twitch token per run.
- **Wikipedia Albums:** `_parse_html_tables()` handles `rowspan` on date `<th>` cells. Last.fm enrichment is per unique primary artist (deduplicated), 200ms rate limit.
- **Export query:** Custom SQL stored in `config.json`. Loaded by `export_sheets._load_custom_query()` at export time. `fetch_rows()` returns `(headers, rows)` ‚Äî column names derived from `result.keys()`, not hardcoded.
